# AeroSpace & Sketchybar

## AeroSpace

[AeroSpace](https://nikitabobko.github.io/AeroSpace/) is an i3-like tiling window manager for macOS. Config is generated by Home Manager at `~/.config/aerospace/aerospace.toml` (source: `modules/home-manager/aerospace.nix`).

### Startup

AeroSpace is launched via a Home Manager `launchd` agent rather than its native `start-at-login`. This avoids the native mechanism writing an absolute `/nix/store` path into macOS Login Items, which would break after every rebuild.

On startup, AeroSpace launches sketchybar and JankyBorders via `after-startup-command`.

## JankyBorders

[JankyBorders](https://github.com/FelixKratz/JankyBorders) draws a colored border around the focused window. Config is generated by Home Manager at `~/.config/borders/bordersrc` (source: `modules/home-manager/jankyborders.nix`).

Current settings: round corners, 6px width, active border `#0088CC`, inactive border `#414550`.

JankyBorders is launched by AeroSpace on startup via `after-startup-command`.

## Sketchybar

[Sketchybar](https://felixkratz.github.io/Sketchybar/) is the status bar. Config lives in `modules/home-manager/sketchybar_config/` and is copied to `~/.config/sketchybar` at build time.

### Fast reload (no rebuild needed)

```bash
rm -rf ~/.config/sketchybar && \
  cp -r ~/nix-darwin-config/modules/home-manager/sketchybar_config/. ~/.config/sketchybar/ && \
  sketchybar --reload
```

The next `darwin-rebuild switch` will overwrite `~/.config/sketchybar` with the nix-managed version.

### Workspace icons

Open windows are shown as app icons using `sketchybar-app-font` glyphs (via `__icon_map` in `plugins/icon_map.sh`). Empty workspaces show a faint ghost icon for their intended purpose.
